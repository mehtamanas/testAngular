Grammar For JQL
Task Page Grid can be filtered using the following Grammars.

A.  For String  : 
    Following Feilds can be used with string operators
          1. Task Name
          2. Assign To
          3. Status
 

   Above Feilds can be filtered using the follwing Operators. There should be space before and after every operator.
                 
    a. "contains" : Syntax  : <Feild> <Operator> <Value>
                    Example : Task Name contains priya
                    Output  : This query will return All the rows where Task Name feild has word priya.
   
    b.  "in"       : Syntax  : <Feild> <Operator> <Value1,Value2...>
                     Example : Task Name in (priya,Rishikesh...) 
                     Output  : This query will return All the rows where "Task Name" feild will have words "priya,Rishikesh.

    c.  "="       : Syntax  : <Feild> <Operator> <Value1>
                    Example : Task Name = priya Gym
                    Output  : This query will return the rows where "Task Name" feild will have exact word "priya Gym"
 
  
   Empty Data or Blank String Data can be filtered using the follwing operators. There should be space before and after every operator.
    
                 
    a.   "is"       Syntax : <Feild> is "blank"
                    Example : Task Name is blank
                    Output : This query will return All the rows where Task Name feild has no data.
                    Note   : "blank" is a keyword defined in our JQL. 
   
    b.    "="       Syntax : <Feild> = "blank"
                    Example : Task Name = blank
                    Output : This query will return All the rows where Task Name feild has no data.
                    Note   : "blank" is a keyword defined in our JQL. 


B.  For Date  :  
     
     1. Due Date 
     2. Created Date 

    Above Feilds can be filtered using the following Operators. There should be space before and after every operator.
    
     a. "isbefore" : Syntax  : <Feild> <Operator> <DD-MM-YYYY> 
                     Example : Due Date isbefore 01-03-2016
                     Output : This query will return All the rows where feild "Due Date" is greater than 01-03-2016.
                     Note : No space between word is and before it is joint word as "isbefore"

     b. "isafter" : Syntax  : <Feild> <Operator> <DD-MM-YYYY> 
                     Example : Due Date isafter 01-03-2016
                     Output : This query will return All the rows where feild "Due Date" is less than 01-03-2016.
                     Note : No space between word is and after it is joint word as "isafter"

     c. "between || " : Syntax  : <Feild> <Operator> <between DD-MM-YYYY || DD-MM-YYYY> 
                     Example : Due Date between 01-03-2016 || 01-04-2016
                     Output : This query will return All the rows where feild "Due Date" is greater than 01-03-2016 or lesser then  01-04-2016.
                     Note : Space is present between words "between ||"

     

    Empty Data or Blank Date Data can be filtered using the follwing operator. There should be space before and after every operator.

     a.  "is":       Syntax  : <Feild> is "never"
                     Example : Due Date is never
                     Output  : This query will return All the rows where feild "Due Date" has no data.
                     Note    : "never" is a keyword defined in our JQL. 

     b.  "=" :       Syntax  : <Feild> = "never"
                     Example : Due Date = never
                     Output  : This query will return All the rows where feild "Due Date" has no data.
                     Note    : "never" is a keyword defined in our JQL. 


    Variables can also be used to filter dates values.
 

    a. "Today"      :    Syntax  : <Feild> is "Today"
                         Example : Due Date is Today
                         Output  : This query will return All the rows where feild "Due Date" is same as current date (Today's date).

   b.  "Yesterday"  : Syntax  : <Feild> is "Yesterday"
                      Example : Due Date is Yesterday
                      Output  : This query will return All the rows where feild "Due Date" is same as yesterday's date. (1 day before current date starting from 00:00 to 23:59)

   c.  "This Week"  : Syntax  : <Feild> is "This Week"
                      Example : Due Date is This Week
                      Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 on most recent Monday to current time.

   d.  "Last Week"  : Syntax  : <Feild> is "Last Week"
                      Example : Due Date is Last Week
                      Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 on Monday before most recent Monday to Sunday 11:59 before most recent Monday.

   e.  "This Month"  : Syntax  : <Feild> is "This Month"
                        Example : Due Date is This Month
                        Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 from 1st day of month to current time.

   f.  "Last Month"  : Syntax  : <Feild> is "Last Month"
                         Example : Due Date is Last Month
                         Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 from 1st of last month to 11:59 on last day of last month.

   g.  "This Quarter"  : Syntax  : <Feild> is "This Quarter"
                         Example : Due Date is This Quarter
                         Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 from 1st of current quarter to current time.

   h.  "Last Quarter"  : Syntax  : <Feild> is "Last Quarter"
                         Example : Due Date is Last Quarter
                         Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 from 1st of last quarter to 11:59 on last day of last quarter. 

   i. "Year to Date"  :  Syntax  : <Feild> is "Year to Date"
                         Example : Due Date is Year to Date
                         Output  : This query will return All the rows where feild "Due Date" has value starting from 00:00 from 1st of this year to current time. 


   j. "This Financial Year"  : Syntax  : <Feild> is "This Financial Year"
                          Example : Due Date is This Financial Year
                      Output  : This query will return All the rows where feild "Due Date" has value starting from Apr 1 of this year to current time. 

   k. "Last Financial Year"  : Syntax  : <Feild> is "Last Financial Year"
                          Example : Due Date is Last Financial Year
                      Output  : This query will return All the rows where feild "Due Date" has value starting from Apr 1 of last year to 11:59 Mar 31 of this year. 
   
   l. "Tomorrow"  :            Syntax  : <Feild> is "Tomorrow"
                          Example : Due Date is Tomorrow
                      Output  : This query will return All the rows where feild "Due Date" has value starting from 1 day after current date starting from 00:00 to 23:59.
   
   m. "Next 7 Days" :          Syntax  : <Feild> is "Next 7 Days"
                          Example : Due Date is Next 7 Days
                      Output  : This query will return All the rows where feild "Due Date" has value starting from 1 day after current date starting from 00:00 to 23:59 to 11:59 of 7th day.



Syntax for Complete "OR" Clause Query

   Task Name = priya or Due Date is last week
   Output : This query will Result where 
                  "Task Name" feild will contain "priya" or 
                  "Due Date" feild has data from monday of this recent week till current time.


Syntax for Complete "AND" Clause Query

   Task Name = priya and Due Date is last week
   Output : This query will Result where 
                  "Task Name" feild will contain "priya" and 
                  "Due Date" feild has data from monday of this recent week till current time.
