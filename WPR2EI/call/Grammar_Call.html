Grammar For JQL
Call Page Grid can be filtered using the following Grammars.

A.  For String  : 
    Following Feilds can be used with string operators
          1. Caller Name
   
   Above Feilds can be filtered using the follwing Operators. There should be space before and after every operator.
                 
    a. "contains" : Syntax  : <Feild> <Operator> <Value>
                    Example : Name contains priya
                    Output  : This query will return All the rows where name feild has word priya.
   
    b.  "in"       : Syntax  : <Feild> <Operator> <Value1,Value2...>
                     Example : Name in (priya,Rishikesh...) 
                     Output  : This query will return All the rows where "Name" feild will have words "priya,Rishikesh.

    c.  "="       : Syntax  : <Feild> <Operator> <Value1>
                    Example : Name = priya Gym
                    Output  : This query will return the rows where "Name" feild will have exact word "priya Gym"
 
  
   Empty Data or Blank String Data can be filtered using the follwing operators. There should be space before and after every operator.
    
                 
    a.   "is"       Syntax : <Feild> is "blank"
                    Example : Name is blank
                    Output : This query will return All the rows where name feild has no data.
                    Note : "blank" is a keyword defined in our JQL. 
   
    b.    "="       Syntax : <Feild> = "blank"
                    Example : Name = blank
                    Output : This query will return All the rows where name feild has no data.
                    Note : "blank" is a keyword defined in our JQL. 


B.  For Date  :  
     
     1. Calldate
    
    Above Feilds can be filtered using the following Operators. There should be space before and after every operator.
    
     a. "isbefore" : Syntax  : <Feild> <Operator> <DD-MM-YYYY> 
                     Example : Calldate isbefore 01-03-2016
                     Output : This query will return All the rows where feild "Calldate" is greater than 01-03-2016.
                     Note : No space between word is and before it is joint word as "isbefore"

     b. "isafter" : Syntax  : <Feild> <Operator> <DD-MM-YYYY> 
                     Example : Calldate isafter 01-03-2016
                     Output : This query will return All the rows where feild "Calldate" is less than 01-03-2016.
                     Note : No space between word is and after it is joint word as "isafter"

     c. "between || " : Syntax  : <Feild> <Operator> <between DD-MM-YYYY || DD-MM-YYYY> 
                     Example : Calldate between 01-03-2016 || 01-04-2016
                     Output : This query will return All the rows where feild "Calldate" is greater than 01-03-2016 or lesser then  01-04-2016.
                     Note : Space is present between words "between ||"

     

    Empty Data or Blank Date Data can be filtered using the follwing operator. There should be space before and after every operator.

     a.  "is":       Syntax  : <Feild> is "never"
                     Example : Calldate is never
                     Output  : This query will return All the rows where feild "Calldate" has no data.
                     Note    : "never" is a keyword defined in our JQL. 

     b.  "=" :       Syntax  : <Feild> = "never"
                     Example : Calldate is never
                     Output  : This query will return All the rows where feild "Calldate" has no data.
                     Note    : "never" is a keyword defined in our JQL. 


    Variables can also be used to filter dates values.
 

    a. "Today"      :    Syntax  : <Feild> is "Today"
                        Example : Calldate is Today
                         Output  : This query will return All the rows where feild "Calldate" is same as current date (Today's date).

   b.  "Yesterday"  : Syntax  : <Feild> is "Yesterday"
                    Example : Calldate is Yesterday
                      Output  : This query will return All the rows where feild "Calldate" is same as yesterday's date. (1 day before current date starting from 00:00 to 23:59)

   c.  "This Week"  : Syntax  : <Feild> is "This Week"
                    Example : Calldate is This Week
                      Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 on most recent Monday to current time.

   d.  "Last Week"  : Syntax  : <Feild> is "Last Week"
                    Example : Calldate is Last Week
                      Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 on Monday before most recent Monday to Sunday 11:59 before most recent Monday.

   e.  "This Month"  : Syntax  : <Feild> is "This Month"
                      Example : Calldate is This Month
                        Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 from 1st day of month to current time.

   f.  "Last Month"  : Syntax  : <Feild> is "Last Month"
                        Example : Calldate is Last Month
                         Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 from 1st of last month to 11:59 on last day of last month.

   g.  "This Quarter"  : Syntax  : <Feild> is "This Quarter"
                        Example : Calldate is This Quarter
                         Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 from 1st of current quarter to current time.

   h.  "Last Quarter"  : Syntax  : <Feild> is "Last Quarter"
                         Example : Calldate is Last Quarter
                         Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 from 1st of last quarter to 11:59 on last day of last quarter. 

   i. "Year to Date"  :  Syntax  : <Feild> is "Year to Date"
                        Example : Calldate is Year to Date
                         Output  : This query will return All the rows where feild "Calldate" has value starting from 00:00 from 1st of this year to current time. 


   j. "This Financial Year"  : Syntax  : <Feild> is "This Financial Year"
                        Example : Calldate is This Financial Year
                      Output  : This query will return All the rows where feild "Calldate" has value starting from Apr 1 of this year to current time. 

   k. "Last Financial Year"  : Syntax  : <Feild> is "Last Financial Year"
                        Example : Calldate is Last Financial Year
                      Output  : This query will return All the rows where feild "Calldate" has value starting from Apr 1 of last year to 11:59 Mar 31 of this year.
