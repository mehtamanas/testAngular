Grammar For JQL - Company


Company Page Grid can be filtered using the following Grammars.

A.  For String  : 
    Following Fields can be used with string operators
       1. Name


   Above Fields can be filtered using the follwing Operators. There should be a space before and after every operator.
                 
    a. "contains" : Syntax  : <Field> <Operator> <Value>
                    Example : Name contains priya
                    Output  : This query will return All the rows where name Field has word priya.
   
    b.  "in"       : Syntax  : <Field> <Operator> <Value1,Value2...>
                     Example : Name in (priya,Rishikesh...) 
                     Output  : This query will return All the rows where "Name" Field will have words "priya,Rishikesh.

    c.  "="       : Syntax  : <Field> <Operator> <Value1>
                    Example : Name = priya Gym
                    Output  : This query will return the rows where "Name" Field will have exact word "priya Gym"

    d.  "!="      : Syntax  : <Field> <Operator> <Value1>
                    Example : Name != priya Gym
                    Output  : This query will return the rows where "Name" Field will not have exact word "priya Gym"
    
   e.   "does not contain" : Syntax  : <Field> <Operator> <Value>
                    Example : Name does not contain priya
                    Output  : This query will return All the rows where name Field does not have word priya.
   
   f.  "not in"     : Syntax  : <Field> <Operator> <Value1,Value2...>
                     Example : Name not in (priya,Rishikesh...) 
                     Output  : This query will return All the rows where "Name" Field will not have words "priya,Rishikesh.


  
   Empty Data or Blank String Data can be filtered using the following operators. There should be space before and after every operator.
    
                 
    a.   "is"       Syntax : <Field> is "blank"
                    Example : Name is blank
                    Output : This query will return All the rows where name Field has no data.
                    Note : "blank" is a keyword defined in our JQL. 
   
    b.    "="       Syntax : <Field> = "blank"
                    Example : Name = blank
                    Output : This query will return All the rows where name Field has no data.
                    Note : "blank" is a keyword defined in our JQL. 


B.  For Date  :  
     
     1. Last Follow Up 

    Above Field(s) can be filtered using the following Operators. There should be space before and after every operator.
    
     a. "is before" : Syntax  : <Field> <Operator> <DD-MM-YYYY> 
                     Example : Last Follow Up Date is before "01-03-2016"
                     Output : This query will return All the rows where Field "Last Follow Up Date" is greater than "01-03-2016".
                    
  THESE OPERATORS CAN BE USED WITH VARIABLES 

    i. "Today" :      Syntax  : <Field> <Operator> Today
                     Example : Last Follow Up Date is before "Today"
                     Output : This query will return All the rows where Field "Last Follow Up Date" has data which is before today i.e returning all the data where last contacted date is till yesterday.

  ii. "Tomorrow" :   Syntax  : <Field> <Operator> Tomorrow
                     Example : Last Follow Up Date is before Tomorrow"
                     Output : This query will return All the rows where Field "Last Follow Up Date" has data which is before Tomorrow i.e returning all the data where last contacted date is till today.
   
     iii. "Yesterday" :   Syntax  : <Field> <Operator> Yesterday
                     Example : Last Follow Up Date is before Yesterday"
                     Output : This query will return All the rows where Field "Last Follow Up Date" has data which is before Yesterday i.e returning all the data where last contacted date is till day before yesterday.

 

     b. "is after" : Syntax  : <Field> <Operator> <DD-MM-YYYY> 
                     Example : Last Follow Up Date isafter 01-03-2016
                     Output : This query will return All the rows where Field "Last Follow Up Date" is less than "01-03-2016".
                  
   THESE OPERATORS CAN BE USED WITH VARIABLES 

    i.    "Today"     : Syntax  : <Field> <Operator> Today
                     Example : Last Follow Up Date is after "Today"
                     Output : This query will return All the rows where Field "Last Follow Up Date" has data which is before today i.e returning all the data where last contacted date is from tomorrow onwards.

  ii. "Tomorrow" :   Syntax  : <Field> <Operator> Tomorrow
                     Example : Last Follow Up Date is after Tomorrow"
                     Output : This query will return All the rows where Field "Last Follow Up Date" has data which is before Tomorrow i.e returning all the data where last contacted date is from day after tomorrow onwards.
   
     iii. "Yesterday" :   Syntax  : <Field> <Operator> Yesterday
                     Example : Last Follow Up Date is after Yesterday"
                     Output : This query will return All the rows where Field "Last Follow Up Date" has data which is before Yesterday i.e returning all the data where last contacted date is from today (i.e after yesterday).

  

     c. "between && " : Syntax  : <Field> <Operator> <between DD-MM-YYYY && DD-MM-YYYY> 
                     Example : Last Follow Up Date between 01-03-2016 && 01-04-2016
                     Output : This query will return All the rows where Field "Last Follow Up Date" is greater than "01-03-2016" or lesser then  "01-04-2016".
                     Note : Space is present between words "between &&"

    a. "Today"      :    Syntax  : <Field> is "Today"
                         Example : Last Follow Up Date is Today
                         Output  : This query will return All the rows where Field "Last Follow Up Date" is same as current date (Today's date).

                      Example : Last Follow Up Date is Yesterday

                      Example : Last Follow Up Date is This Week
                      Output  : This query will return All the rows where Field "Last Follow Up Date" has value starting from 00:00 on most recent Monday to current time.

                      Example : Last Follow Up Date is Last Week
                      Output  : This query will return All the rows where Field "Last Follow Up Date" has value starting from 00:00 on Monday before most recent Monday to Sunday 11:59 before most recent Monday.

   e.  "This Month"  : Syntax  : <Field> is "This Month"
                        Example : Last Follow Up Date is This Month
                        Output  : This query will return All the rows where Field "Last Follow Up Date" has value starting from 00:00 from 1st day of month to current time.

   f.  "Last Month"  : Syntax  : <Field> is "Last Month"

                         Output  : This query will return All the rows where Field "Last Follow Up Date" has value starting from 00:00 from 1st of last month to 11:59 on last day of last month.

   g.  "This Quarter"  : Syntax  : <Field> is "This Quarter"
                         Example : Last Follow Up Date is This Quarter
                         Output  : This query will return All the rows where Field "Last Follow Up Date" has value starting from 00:00 from 1st of current quarter to current time.

   h.  "Last Quarter"  : Syntax  : <Field> is "Last Quarter"
                         Example : Last Follow Up Date is Last Quarter
                         Output  : This query will return All the rows where Field "Last Follow Up Date" has value starting from 00:00 from 1st of last quarter to 11:59 on last day of last quarter.

     3. QUOTE
     4. RATING

     Above Fields can be filtered using the following Operators. There should be space before and after every operato     

  a. ">="  : Syntax  : <Field> <Operator> <Value>
                   Example : PEOPLE >= 5
                   Output : This query will return All the rows where Field "PEOPLE" is greater then equal (" >= ") to 5.

 
b. "<="  : Syntax  : <Field> <Operator> <Value>
                    Example : PEOPLE <= 5
                    Output : This query will return All the rows where Field "PEOPLE" is less then equal (" <= ") to 5.
  
 
c.  "="   : Syntax  : <Field> <Operator> <Value1>
                    Example : PEOPLE = 1
                    Output : This query will return All the rows where Field "PEOPLE" is equal (" = ") to 1.

         
        d.  ">"    : Syntax  : <Field> <Operator> <Value>
                     Example : PEOPLE > 5
                     Output : This query will return All the rows where Field "PEOPLE" is greater then (" > ") to 5.

       
e.  "<"    : Syntax  : <Field> <Operator> <Value>
                     Example : PEOPLE < 5
                     Output : This query will return All the rows where Field "PEOPLE" is less then (" < ") to 5.



Order by Clause  : Here grid can be arranged either in ascending order or desending order using the following Syntax.

    a. "asc"  :      Syntax    : Order by <Field> asc
                     Example 1 : Order by people asc
                     Output    : This query will rearrange the grid with "PEOPLE" Field in ascending order.

    Example 2 : PEOPLE >= 5 Order by PEOPLE asc
                     Output    : This query will return all the rows where PEOPLE >= 5 and then those rows it will rearrange with "PEOPLE" Field in ascending order.


     a. "desc"  :    Syntax    : Order by <Field> desc
                     Example 1 : Order by people desc
                     Output    : This query will rearrange the grid with "PEOPLE" Field in descending order.

    Example 2 : PEOPLE >= 5 Order by people desc
                     Output    : This query will return all the rows where PEOPLE >= 5 and then those rows it will rearrange with "PEOPLE" Field in descending order.

Name contains priya or Email in (burzin@dwellar.com,pradip@dwellar.com) or people = 6

                   or PEOPLE = 6

Name contains priya and Email in (burzin@dwellar.com,pradip@dwellar.com) and people = 6

                  "Name" Field should contain "priya" and 
                  "Email" Field should contain "burzin@dwellar.com" and "pradip@dwellar.com"  
                   and PEOPLE should be = 6
