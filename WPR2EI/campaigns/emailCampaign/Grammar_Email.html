Grammar For JQL
Email Page Grid can be filtered using the following Grammars.

A.  For String  : 
    Following Feilds can be used with string operators
             1. Name
             2. Status  

   Above Feilds can be filtered using the follwing Operators. There should be space before and after every operator.
                 
    a. "contains" : Syntax  : <Feild> <Operator> <Value>
                    Example : Status contains Running
                    Output  : This query will return All the rows where Status feild has word Running.
   
    b.  "in"       : Syntax  : <Feild> <Operator> <Value1,Value2...>
                     Example : Status in (Running,Completed,...) 
                     Output  : This query will return All the rows where "Status" feild will have words "Running,Completed....

    c.  "="       : Syntax  : <Feild> <Operator> <Value1>
                    Example : Status = Running
                    Output  : This query will return the rows where "Status" feild will have exact word "Running"
 
  
   Empty Data or Blank String Data can be filtered using the follwing operators. There should be space before and after every operator.
    
                 
    a.   "is"       Syntax : <Feild> is "blank"
                    Example : Status is blank
                    Output : This query will return All the rows where Status feild has no data.
                    Note   : "blank" is a keyword defined in our JQL. 
   
    b.    "="       Syntax : <Feild> = "blank"
                    Example : Status = blank
                    Output : This query will return All the rows where Status feild has no data.
                    Note   : "blank" is a keyword defined in our JQL. 


B. For Numbers  : 
         
                 1. OPEN RATE
                 2. OPENS
                 3. BOUNCES
                 4. UNSUBSCRIBES
                5. BUDGET
                 6. REVENUE 
                 7. CONVERSIONS
                 8. CLICKS

     Above Feilds can be filtered using the following Operators. There should be space before and after every operato     

  a. ">="  : Syntax  : <Feild> <Operator> <Value>
                   Example : Open Rate >= 5
                   Output : This query will return All the rows where feild "Open Rate" is greater then equal (" >= ") to 5.

 
b. "<="  : Syntax  : <Feild> <Operator> <Value>
                    Example : Open Rate <= 5
                    Output : This query will return All the rows where feild "Open Rate" is less then equal (" <= ") to 5.
  
 
c.  "="   : Syntax  : <Feild> <Operator> <Value1>
                    Example : Open Rate = 1
                    Output : This query will return All the rows where feild "Open Rate" is equal (" = ") to 1.

         
        d.  ">"    : Syntax  : <Feild> <Operator> <Value>
                     Example : Open Rate > 5
                     Output : This query will return All the rows where feild "Open Rate" is greater then (" > ") to 5.

       
e.  "<"    : Syntax  : <Feild> <Operator> <Value>
                     Example : Open Rate < 5
                     Output : This query will return All the rows where feild "Open Rate" is less then (" < ") to 5.



Order by Clause  : Here grid can be arranged either in ascending order or desending order using the following Syntax.

    a. "asc"  :      Syntax    : Order by <Feild> asc
                     Example 1 : Order by Open Rate asc
                     Output    : This query will rearrange the grid with "Open Rate" Feild in ascending order.

    Example 2 : Budget >= 5000 Order by Open Rate asc
                     Output    : This query will return all the rows where budget >= 5000 and then those rows it will rearrange with "Open Rate" Feild in ascending order.


     a. "desc"  :    Syntax    : Order by <Feild> desc
                     Example 1 : Order by Open Rate desc
                     Output    : This query will rearrange the grid with "Open Rate" Feild in descending order.

    Example 2 : Budget >= 5000 Order by Open Rate desc
                     Output    : This query will return all the rows where budget >= 5000 and then those rows it will rearrange with "Open Rate" Feild in descending order.
